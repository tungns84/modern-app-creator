<feature>
  <name>Project rename engine (com.acme.app → team values)</name>
  <files>scripts/init-core.mjs, scripts/tests/init-core.test.mjs</files>
  <behavior>
    Operating on a fixture tree in a temp dir:
    - validateInputs: groupId must match Java package grammar `^[a-z][a-z0-9]*(\.[a-z][a-z0-9_]*)*$`; artifactId must match `^[a-z][a-z0-9-]*$`; violations → {ok:false, errors naming the field}; valid → {ok:true}
    - Dirty worktree (uncommitted fixture change) → main refuses with exit code 1 and message containing "dirty" BEFORE any file is modified
    - planRenames on a tree containing `src/com/acme/app/App.java` with content `package com.acme.app;` + a `pom.xml` with `acme-app` → fileEdits cover both files, dirRenames maps `com/acme/app` → `com/example/demo` (for --group-id com.example --artifact-id demo)
    - Word-boundary safety: content `com.acme.application` is NOT rewritten (substring guard, Pitfall 8)
    - Exclusions: fixture files under `.planning/`, `product/`, `requirements/`, `.git/`, and a fixture `logo.png` (binaryGlobs per PRESET-SPEC: **/*.png, **/*.svg, **/*.jar, **/mvnw*) appear in NO fileEdits even when they contain the literal
    - Idempotency: after applyRenames, a second planRenames returns occurrenceCount 0 and main exits 0 printing "already initialized as demo"
    - applyRenames + main produce exactly one new git commit with message `chore: initialize project as demo`
  </behavior>
  <implementation>
    Zero npm dependencies (node:fs, node:path, node:child_process for git). NO `sed -i` anywhere (macOS/Linux divergence, Pitfall 8). Literal family read from scripts/init-replacements.json so Giai đoạn 2 templating reuses the manifest. Word-boundary check: character after the literal must not be [a-zA-Z0-9] continuation (e.g., reject match when followed by "lication"). Entry points are thin: init.sh = `#!/bin/sh` + `exec node "$(dirname "$0")/init-core.mjs" "$@"`; init.ps1 = `node "$PSScriptRoot/init-core.mjs" @args` — parity reduces to argument passing (D-23 entry-point reading per Pitfall 8). init-core also performs the go-task pinned-version check (warn if `task` missing or version != v3.51.1 — root CLAUDE.md constraint, RESEARCH install note).
  </implementation>
</feature>
