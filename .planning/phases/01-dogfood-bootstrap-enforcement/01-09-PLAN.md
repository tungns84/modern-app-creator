<feature>
  <name>Plan-compliance verdict (GATE-10 logic core)</name>
  <files>scripts/checks/plan-compliance.mjs, scripts/checks/assert-non-author-approval.mjs, scripts/checks/tests/plan-compliance.test.mjs</files>
  <behavior>
    evaluate() over fixture inputs — one fixture pair (files, reviews) per case:
    - Changed files all non-T3 (e.g., src/foo.ts, README.md) → PASS with reason "no T3 paths touched"
    - Changed files only under .planning/** → PASS (meta exclusion via shared matcher, D-06)
    - T3 file touched (.github/workflows/x.yml), no specs/NNN-*/plan.md in specsTree → FAIL, reason names the touched path and the missing artifact
    - T3 touched, plan.md exists but lacks `tier: T3` or lacks `Approved-by:` → FAIL naming the missing marker
    - T3 touched, plan present+approved, reviews contain one APPROVED by login != author, user.type "User" → PASS
    - Reviews contain APPROVED then later CHANGES_REQUESTED by the same user → that user does NOT count (latest-per-user reduction, Pitfall 3)
    - Only review is APPROVED with user.type "Bot" → identity step fails (bot exclusion)
    - Only approval is by the PR author → identity step fails (no self-approval)
    - Identity step failed BUT waivers contain non-expired {scope:"self-approval"} → PASS with warning containing the waiver id (D-10)
    - Same waiver with expires < now → FAIL, reason "waiver expired" (waiver-rot, Security table)
    - reduceReviews: chronological list reduced to latest per login; only state APPROVED survives; DISMISSED never counts
  </behavior>
  <implementation>
    Pure verdict core: evaluate() takes plain data, no network — the workflow shell fetches via `gh api` (pulls/N/files --paginate, pulls/N/reviews --paginate, pulls/N for author) and feeds JSON files to the CLI (Pitfall 5: derive PR number once from the event, fetch everything fresh). Import matchTier/loadTiers from .claude/hooks/lib/tiers.mjs — never a second matcher (D-22). Copy the trigger block and jq reduction exactly from the Q-002 spike report. Failure output: one line per reason naming the violated rule + the fix (FR-E08 contract). Workflow token: permissions contents: read, pull-requests: read.
  </implementation>
</feature>

| T-01-26 | Spoofing | Bot approval satisfying gate | mitigate | user.type Bot exclusion, fixture-tested |
