    Decision function over (stdin payload, tiers.json, current git branch, specs tree). Cases (10):
    - Write payload `{tool_name:"Write", tool_input:{file_path:".github/workflows/x.yml"}}` on an UNBOUND branch (e.g., main — no feat/NNN-* match) → deny JSON, reason contains "specs/", "plan", the branch convention "feat/NNN-" and a next step
    - Same payload on branch `feat/099-anything` where specs/ contains ONLY unrelated approved specs (e.g., specs/001-x/plan.md with Approved-by) → deny JSON naming the missing BOUND artifact `specs/099-*/plan.md` — unrelated approved specs never satisfy the gate (anti-rot, checker blocker)
    - Same payload on branch `feat/001-x` with `specs/001-x/plan.md` containing `Approved-by: <name>` → no decision (exit 0, empty stdout)
    - Write to `.planning/notes.md` → no decision (meta exclusion, D-06)
    - Write to `src/whatever.ts` (no T3 match) → no decision
    - Bash payload `{tool_name:"Bash", tool_input:{command:"git push --force origin main"}}` → deny JSON, reason contains "human"
    - Bash payload `command:"task verify"` → no decision
    - Bash payload `command:"echo hacked > .cowork/tiers.json"` (write-bypass into T3 path) without a bound approved plan → deny JSON (best-effort scan: redirects `>`/`>>`, `tee`, `sed -i`, `git apply` targeting T3-matched paths)
    - Malformed stdin (not JSON) where script cannot prove path is safe → deny JSON containing "failing closed" guidance (Pitfall 1)
    - matchTier glob subset: `**` crosses dirs, `*` within segment, literal paths; `.cowork/**` matches `.cowork/tiers.json`; `**/pom.xml` matches `backend/pom.xml`; specNumberFromBranch: `feat/007-x` → "007", `main`/`chore/cleanup` → null
