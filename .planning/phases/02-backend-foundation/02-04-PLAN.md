|----------|-------------|
| MDC context → structured logs | request-scoped MDC values cross into persisted/forwarded log output; PII must be filtered here |
| app → Actuator endpoints | liveness/readiness/prometheus/metrics endpoints expose runtime state to scrapers/orchestrators |
| build → main branch | new module boundary + pom dependency (logstash-logback-encoder) + logback config are T3 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Information disclosure | PII (email/ip/name) leaking into structured JSON logs via MDC | mitigate | `LogstashEncoder` explicit `includeMdcKeyName` allowlist — only listed keys emit; `MdcAllowlistTest` proves non-allowlisted keys are dropped; allowlist sourced from `ObservabilityProperties` |
| T-02-11 | Information disclosure | Actuator endpoints over-exposed (env values, heapdump, full metrics) to unauthenticated scrapers | mitigate | `management.endpoints.web.exposure.include` limited to health,prometheus,info,metrics; `env.show-values=NEVER` (Boot default) asserted; endpoint auth/scoping hardening is Phase 3 security scope (noted) |
| T-02-12 | Information disclosure | OTLP trace export leaking sensitive span attributes off-host | accept | Local dev exports to the in-cluster lgtm container only; span-attribute scrubbing + prod OTLP TLS is deferred to deploy hardening (Phase 6) — dev/seam scope, low risk |
| T-02-13 | Tampering | Cleanup task deleting non-completed (still-needed) event records → silent event loss | mitigate | Cleanup targets only COMPLETED records older than retention via the Modulith completed-publications API (never INCOMPLETE); Plan 05 kill-listener test proves no-loss holds with cleanup active |
| T-02-SC | Tampering | npm/Maven installs (logstash-logback-encoder added to pom) | mitigate | Single well-known Apache-2.0 encoder pinned to a confidence-labelled stable version in the bound spec; reviewed at H2; no transitive runtime surprises (logging only) |
</threat_model>
