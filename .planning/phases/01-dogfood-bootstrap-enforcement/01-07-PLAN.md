<feature>
  <name>CLAUDE.md render-then-check pipeline (GATE-12 / D-14)</name>
  <files>scripts/checks/render-claude-md.mjs, scripts/checks/claude-md-check.mjs, scripts/checks/tests/claude-md.test.mjs</files>
  <behavior>
    - render("Hello {{projectName}}", {projectName:"smoke"}) → "Hello smoke"
    - render with a value missing for a used placeholder → throws naming the placeholder (orphan `{{` after render = fail, PRESET-SPEC §4.5)
    - render never touches content without `{{ }}` tokens (token replacement only — no other transforms)
    - countLines("a\r\nb\r\nc") === countLines("a\nb\nc") === 3 (CRLF normalization, Pitfall 6)
    - checkBudget on a 201-line fixture with max 200 → {pass:false, actual:201}; the failure message names the file, the budget, and the actual count
    - runSmoke with a manifest declaring 2 commands for the current leg executes them and reports failures by command; runSmoke with a manifest declaring 0 commands for the leg → throws "no commands declared" (vacuous-green guard, Pitfall 9)
    - Smoke manifest format: JSON `{commands:[{cmd:"task --list", legs:["ubuntu","windows","macos"]}]}` — commands filtered by `legs` so Docker-needing commands restrict to ubuntu (Pitfall 9)
  </behavior>
  <implementation>
    Zero npm dependencies. render core ~20 lines (D-14 / PRESET-SPEC §4 token-replacement-only — matches the Giai đoạn 2 generator contract). claude-md-check composes: render the root template with sample values (projectName=smoke, groupId=com.acme, artifactId=app) to a temp file → checkBudget ≤200; checkBudget each --tree-file ≤150; runSmoke the manifest. Every failure line names the violated rule + file + measured value (deny-message UX contract from PATTERNS Shared Patterns). Exit code = number of failed rules capped at 1.
  </implementation>
</feature>
